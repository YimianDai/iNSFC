# iNSFC 2026 - 国家自然科学基金 LaTeX 模板 (非官方)

## 写在前面

这是本模板的第三次更新了。说来也巧，第一次做这个模板，是受佳伟之托帮他申报国自然时搞一个 LaTeX 模板，说在南开程明明老师的版本上修改。没想到兜兜转转，博后出站后我竟然机缘巧合入职南开加入了程老师团队，这也是一种缘分。今年正好也是我申请面上，干脆一鼓作气把它从底到外再翻新一遍，把以前想加没加的功能全做进去。个人水平终究有限，**欢迎各位老师批评指正**！

本项目是一套结构清晰、配置统一的 NSFC 申请书 LaTeX 模板集合。针对 2026 年最新官方 Word 模板进行了逐字逐句的校验与适配。我们修复了原版中部分格式问题，并**新增了“国家重大科研仪器研制项目”正文模板**，旨在帮助申请人从繁琐的格式排版中解脱出来，专注于内容的逻辑与创新。



**特别致谢：**
在这里着重致谢 **OnlineLaTeX** 平台上的版本。本项目的基础源码，正是基于他们的优秀工作进行二次开发与集成的。

## 这次升级，到底更新了什么？

这次更新的核心逻辑在于：**在尽可能贴合官方形式审查要求、降低废标风险的前提下，并进一步降低排版的使用门槛。** 有时候代码的“优雅”不得不向官方设定的审查标准妥协。

以下是具体的更新内容：

### 一、 填坑与功能补全

1. **补上了重大仪器模板**：之前开源环境里基本找不到这个本子的 LaTeX 模板。它的提纲多了验收指标、研制风险等几个特殊章节，这次直接对照官方 Word 复刻，补齐这块空白。
2. **添加预算说明书**：官方那个带边框的预算书用常规代码较难排版。我们使用调了 `mdframed` 宏包重绘边框，把官方要求的那种“单倍行距+段后8磅”、以及填报区和说明区字体的来回切换给硬抠出来了。
3. **对齐官方Word不规则排版**：官方 Word 里本来就有不少不规范的空格和莫名其妙的不加粗（如前括号不加粗但是后括号又加粗了）。为了尽量防初筛出问题，我们在代码里把Word中这些反直觉的格式（比如强行 `\linebreak` 换行）全保留了，使得每一行都能与官方word对应。

### 二、 重构代码逻辑

1. **配置与正文分开**：排版逻辑全放到了 `config.tex`，把常用的视觉开关（比如编号风格、字体切换）放进了 `style.tex`，让正文文件清爽一点，修改格式只需这两个配置文件。
2. **封装官方说明**：统一封装成了 `\NsfcHeader` 和 `\NsfcSubHeader`。防止误改官方格式，而且保证全文字体字号一致。
3. **集成各类文档**：合并了面上、重大仪器、重点、地区和青A/B/C的入口

### 三、 排版细节与编译环境

1. **优化中文字体加粗**：LaTeX 默认的中文字体加粗发虚，跟 Word 算法加粗的浓淡对不上。这次用 `xeCJK` 的 `AutoFakeBold=1.8` 拉高了楷体和仿宋的字重，宋体粗体直接拿思源宋体（`siyuansongti-Bold.otf`）顶替，这种加粗效果比默认的伪加粗看着更加突出。
2. **拆分各项目的页边距**：各类本子的官方 Word 页边距和缩进其实是不一样的。这次我们使用官方 Word 的实际参数，给每个 `.tex` 入口单独设了对应的 `\geometry` 和缩进变量。
3. **预算书字体**：预算书里，官方说明（仿宋）和用户填报（宋体）混在一起很容易互相污染格式。这次单独写了个局部样式宏，把仿宋字体和行距锁死在说明区里，互不干涉。
4. **自带字体库，云端免配环境**：避免本地电脑因为缺字体报错。用到的核心字体在 `fonts/` 文件夹里。

## 快速开始

### 1. 编译

* **最稳妥方案：直接上 Overleaf（强烈推荐）**

1. 把整个项目打个 `.zip` 压缩包。
2. 扔进 Overleaf 选 Upload Project 创建项目。
3. 点开左上角 `Menu`，找到 `Compiler`，必须切成 `XeLaTeX`
4. 打开你要写的本子（比如 `面上.tex`），点 Recompile 。

* **本地编译（VSCode / TeXStudio 等）**
编译引擎为 `XeLaTeX`。标准编译链是：`XeLaTeX -> BibTeX -> XeLaTeX -> XeLaTeX`。

### 2. 全局样式修改

不需要在正文文件里改颜色和字体。在 `style.tex` 里改变量：

* **正文字体 (`\UseKaitiBody`)**：填 `false` 就是标准的宋体正文。如果嫌刺眼或者老板有偏好，改成 `true` 一键切成柔和的楷体。
* **标题字体 (`\HeadingFont`)**：统管一二级标题的字体，支持在黑体、楷体、宋体之间切换。
* **标题编号风格 (`\NumberingStyle`)**：看个人习惯，直接填数字 1、2 或 3：
* `1` = 1 / 1.1 / 1.1.1 
* `2` = 1. / 1.1 / 1.1.1 （一级标号后带点）
* `3` = (一) / 1. / (1) 


* **超链接与引用颜色 (`\ThemeColor`)**：控制参考文献、公式标号的颜色。

底层的排版逻辑在 `config.tex`，正常情况尽量不要修改，如需修改请仔细看注释，了解代码段功能。

## 避坑指南

1. **不要动蓝色提纲**：代码里用 `\NsfcHeader` 等宏生成的蓝色标题和说明文字，**绝对不要改**。避免形式审查出问题。
2. **别删那些“丑代码”**：源码里如果你看到奇奇怪怪的强制换行（比如 `\linebreak` 或 `\\`）和空格，不要去删它们。那都是为了硬凑官方 Word 的对齐方式特意留下的，删了反而跟官方排版对不上了。
3. **卡死页数红线**：一些本子官方明确标注了“原则上不超过 30 页”或是限制字数，不要超页/字数。

## 快速开始

由于本项目依赖特定的系统中文字体（宋体、楷体），本地编译环境极易因字体库缺失或文件名不匹配导致报错。

### 方案 A：Overleaf 在线编译（推荐）

**无需配置本地环境，上传即用**

1. 将本项目文件夹打包为 ZIP 压缩包。
2. 在 [Overleaf](https://www.overleaf.com) 创建新项目 -> `Upload Project`。
3. **关键设置**：点击左上角 `Menu` -> `Compiler` -> 选择 **`XeLaTeX`** (必须选这个，否则不通)。
4. 打开任意 `.tex` 文件（如 `面上.tex`）点击 Recompile 即可。

### 方案 B：本地编译

如果您坚持在本地（VSCode + LaTeX Workshop, TeXStudio 等）编译，请确保：

1. 安装 **TeX Live** (建议 2024 或更高版本)。
2. 编译引擎必须设置为 `XeLaTeX`。
3. **字体依赖**：系统必须能识别项目根目录 `fonts/` 文件夹下的字体，或者您需要手动安装这些字体到操作系统中（特别是 Windows 以外的系统）。
4. **编译链**：`XeLaTeX -> BibTeX -> XeLaTeX -> XeLaTeX`。

## ⚠️ 警告与注意事项

为了确保您的申请书不因格式问题被初筛掉，请务必遵守以下规则：

1. **严禁修改蓝色提纲文字**：
* 模板中蓝色的提纲标题（如 `（一）立项依据`）及括号内的说明文字，**一个标点符号都不要改**。
* 本模板已根据 2026 官方 Word 锁定了这些文字，请勿触碰。


2. **请勿删除改动代码中的“奇怪”空格**：
* 在 `.tex` 源代码中，您会看到诸如 `\hspace{2em}` 或强制换行 `\\` 的代码。
* **这是为了强行对齐官方 Word 中的排版**（官方 Word 经常出现对不齐的情况）。为了视觉上与 Word 一致，请保留这些代码，不要为了代码美观而删除它们。


3. **页数限制**：
* **青年项目（C类）**、**面上项目**：官方建议正文原则上不超过 **30页**。
* **青A/青B**：各章节有明确的字数建议（如 4000 字、8000 字），请留意注释。


## 配置与自定义

配置集成在了 `config.tex`，大多数情况下不需要修改这个文件，但以下情况除外：

* **修改页边距**：每个 `.tex` 文件的头部都有独立的 `\geometry` 设置。这是因为不同学部的 Word 模板页边距竟然是不一样的（例如重大仪器和面上项目的边距微有不同），我们已预设好，如需微调请直接修改对应 `.tex` 文件头部。
* **参考文献格式**：默认使用 `gbt7714-nsfc.bst`。如需修改引用格式（如从 `[1]` 改为 `(Author, Year)`），请修改 `config.tex` 中的 `natbib` 设置。

## Q&A 常见问题

**Q: 编译报错 `font not found`？**

**A:** 这是最常见的问题。请检查：1. 您的编译引擎是否为 `XeLaTeX`？2. `fonts/` 目录下是否真实存在字体文件？**最快解决方案是转用 Overleaf。**

**Q: 这里的“重大仪器”模板可以直接用吗？**

**A:** 可以。我们特别比对了 2026 版重大仪器 Word 模板，修正了与面上项目不同的提纲结构（例如增加了“验收指标”和“研制风险”章节），且保留了官方模板中“第四部分标题加粗不统一”的特性，当然还是建议您再与官方模板对比一下，防止出现不一致的地方。

**Q: 生成的 PDF 还需要转 Word 吗？**

**A:** **不需要**。NSFC 系统支持（且推荐）直接上传 PDF 格式的正文文件。LaTeX 生成的 PDF 排版更稳定，不会出现 Word 换台电脑格式就乱了的情况。
