## 本次升级的核心更新

这次更新的核心逻辑在于：**在尽可能贴合官方形式审查要求、降低废标风险的前提下，进一步降低排版的使用门槛。** 我们主动让代码的“优雅”向官方设定的审查标准妥协。

### 一、 功能补全（本次新增）

1. **补上重大仪器模板**：之前开源环境里基本找不到这个本子的 LaTeX 模板。它的提纲多了验收指标、研制风险等几个特殊章节，这次直接对照官方 Word 复刻，补齐空白。
2. **添加预算说明书**：官方带边框的预算书用常规代码较难排版。我们使用 `mdframed` 宏包重绘边框，硬抠出了官方要求的“单倍行距+段后8磅”，以及填报区和说明区字体的无缝切换。
3. **对齐官方Word不规则排版**：官方 Word 中存在不少不规律的空格和莫名其妙的不加粗（如前括号不加粗后括号加粗）。为尽可能还原，我们在代码里保留了这些反直觉的格式（如强行 `\linebreak` 换行），确保每一行都能与官方高度对应。
4. **自定义加粗**：新增自定义加粗指令，可自定义加粗颜色，使高亮内容更符合个人排版风格。

### 二、 架构重构与底层优化（本次重构）

1. **配置与正文解耦**：排版逻辑全放到了 `config.tex`，常用的视觉开关（编号风格、字体切换）放进 `style.tex`。修改格式只需动这两个配置文件，保持正文清爽。
2. **封装官方说明**：统一封装成 `\NsfcHeader` 和 `\NsfcSubHeader`，防止误改官方格式，保证全文字体字号一致。
3. **集成各类文档**：合并了面上、重大仪器、重点、地区和青A/B/C的项目入口。
4. **优化中文字体加粗**：LaTeX 默认的中文字体加粗发虚。这次用 `xeCJK` 的 `AutoFakeBold=1.8` 拉高了楷体和仿宋的字重，其中宋体粗体使用思源宋体顶替，并引入缩放因子抹平不同字体的视觉大小差异，使加粗效果更实、更突出。
5. **拆分各项目的页边距**：根据官方 Word 的实际参数，给每个 `.tex` 入口单独设置了对应的 `\geometry` 和缩进变量。
6. **局部字体锁定**：针对预算书，单独编写局部样式宏，将官方说明（仿宋）和用户填报（宋体）的字体与行距严格隔离，防止互相污染。
7. **自带字体库**：内置核心中文字体于 `fonts/` 文件夹，云端免配环境，解决跨平台编译报错问题。

## 项目文件树

建议在使用前先大致了解项目的基本骨架，日常使用主要关注 `style.tex` 和 所写的主文件 即可。推荐使用者自行新建章节文件夹（section）📂并在正文新建引用 `\input...` 方便撰写。

```text
NSFC_正文模板/
├── style.tex              # 【用户操作区】排版风格开关（字体、编号、颜色等）
├── config.tex             # 【底层逻辑区】全局环境宏包与专用宏定义（非必要建议勿动）
├── ref.bib                # 参考文献数据库
├── gbt7714-nsfc.bst       # 符合国标的参考文献样式（源自onlinelatex版本）
├── fonts/                 # 核心中文字体库（跨平台编译不报错的基础保障）
│
├── 面上.tex / 重点.tex / 地区.tex    # 常规项目入口
├── 重大仪器.tex                      # [新增] 重大科研仪器项目入口
├── 青A.tex / 青B.tex / 青C.tex       # 优青 / 海外优青 / 普青入口
│
├── 预算说明书（重大）.tex             # [新增] 重大项目预算表
└── 预算说明书（除重大）.tex           # [新增] 常规项目预算表

```

## 模板使用指南

### 1. 编译环境配置

* **最稳妥方案：直接上 Overleaf（强烈推荐）**
1. 把整个项目打个 `.zip` 压缩包。
2. 扔进 Overleaf 选 Upload Project 创建项目。
3. 点开左上角 `Menu`，找到 `Compiler`，必须切成 `XeLaTeX`。
4. 打开你要写的本子（比如 `面上.tex`），点 Recompile。


* **本地编译（VSCode / TeXStudio 等）**
编译引擎为 `XeLaTeX`。标准编译链是：`XeLaTeX -> BibTeX -> XeLaTeX -> XeLaTeX`。

### 2. 全局样式调整

不需要在正文文件里改颜色和字体。所有视觉配置均在 `style.tex` 中通过修改变量完成：

* **正文字体 (`\UseKaitiBody`)**：填 `false` 为标准宋体正文。若需柔和视觉效果，改为 `true` 一键切成楷体。
* **标题字体 (`\HeadingFont`)**：统管一二级标题字体，支持黑体、楷体、宋体切换。
* **标题编号风格 (`\NumberingStyle`)**：填数字 1、2 或 3：
* `1` = 1 / 1.1 / 1.1.1
* `2` = 1. / 1.1 / 1.1.1 （一级标号后带点）
* `3` = (一) / 1. / (1)


* **超链接与引用颜色 (`\ThemeColor`)**：控制参考文献、公式标号的颜色。



### 3. 布局调控与自定义强调

以下参数用于控制文稿的“呼吸感”与视觉重点，是应对评审专家视觉疲劳、或在极限情况下控制篇幅的利器：


| 控制维度 | 参数命令 | 推荐取值与排版策略 |
| :--- | :--- | :--- |
| **段间距** | `\UserParskip` | **`0.5ex`**：默认值，视觉适中。<br>**`0pt` 或 `0.1ex`**：瘦身策略，在不删减内容的前提下极限压缩篇幅。 |
| **列表模式** | `\UserListMode` | **`0` (极致紧凑)**：列表内外无额外空隙。适合指标项繁多、需极力节省空间的场景。<br>**`1` (宽松美观)**：随段间距自动缩放，排版具学术美感，适合充实版面。 |

**局部视觉强调：自定义颜色加粗 (`\mybf`)**

这是对标准 `\textbf` 的增强。在正文中直接调用 `\mybf{核心内容}`，不仅会对文字加粗，还会将其渲染为深邃的**学术蓝 (#000040)**,也可以改成自己喜欢的颜色。

* **实战用途**：专用于立项依据或创新点中的核心指标与关键词抓取。该颜色打印时接近纯黑（合规且不死板），电子档阅卷时则能极大地缓解视觉疲劳，比纯黑加粗更能精准引导评审视线。

*(注：底层的排版逻辑在 `config.tex`，正常情况尽量不要修改。如需修改请仔细看注释，了解代码段功能。)*

## 避坑指南

1. **绝对不要动蓝色提纲**：代码里用 `\NsfcHeader` 等宏生成的蓝色标题和说明文字是基金委的红线，**绝对不要改**，避免形式审查被毙。
2. **不要改那些“丑代码”**：源码里如果你看到奇奇怪怪的强制换行（如 `\linebreak` 或 `\\`）和空格，不要去删。这是为了硬凑官方 Word 的对齐方式特意留下的，删了反而跟官方排版脱节。
3. **卡死页数红线**：一些本子官方明确标注了“原则上不超过 30 页”或是限制字数，请严格遵守，不要超限。
